# Project Structure Detailed Guide

This file explains every major folder and file in the repository and what each one does.

## Root

- `README.md`
  - Master assignment guide.
  - Defines project objective, phases, deliverables, and workflow.

- `.venv/`
  - Python virtual environment used for automation scripts and UI.
  - Contains installed Python packages used by scripts.

- `test_O0`, `test_O2`
  - Demo binaries built from `testcases/ub_signed_overflow.c`.
  - Used to show optimization behavior differences.

## Benchmarks

- `testcases/`
  - UB-focused C benchmark programs used for evaluation.

- `testcases/ub_signed_overflow.c`
  - Signed integer overflow UB benchmark.

- `testcases/ub_null_deref.c`
  - Null pointer dereference UB benchmark.

- `testcases/ub_oob_access.c`
  - Out-of-bounds array access UB benchmark.

- `testcases/ub_invalid_shift.c`
  - Invalid shift UB benchmark (negative / too-large shift).

- `testcases/ub_use_after_free.c`
  - Use-after-free UB benchmark.

- `testcases/ub_uninitialized.c`
  - Uninitialized read UB benchmark.

- `testcases/ub_div_zero.c`
  - Integer division-by-zero UB benchmark.

- `testcases/ub_type_punning.c`
  - Strict aliasing/type punning UB benchmark.

- `testcases/ub_double_free.c`
  - Double free UB benchmark.

- `testcases/ub_string_literal.c`
  - String literal modification UB benchmark.

- `testcases/safe_programs/`
  - Safe baseline programs used for false-positive checks.

- `testcases/safe_programs/safe_arithmetic.c`
  - Safe integer handling baseline.

- `testcases/safe_programs/safe_array.c`
  - Safe bounds-checked array access baseline.

- `testcases/safe_programs/safe_pointers.c`
  - Safe pointer/null-check baseline.

## LLVM Experiments

- `llvm_experiments/`
  - Stores optimization experiment artifacts.

- `llvm_experiments/results/`
  - Generated per benchmark:
  - `*_O0.ll`: LLVM IR at O0.
  - `*_O2.ll`: LLVM IR at O2.
  - `*_O0.s`: assembly at O0.
  - `*_O2.s`: assembly at O2.
  - `*_diff.txt`: textual diff between O0 and O2 IR.

## Tool Outputs

- `tool_results/`
  - Logs generated by static/runtime analysis tools.

- `tool_results/ubsan_results.txt`
  - Runtime UB findings from UBSan/ASan instrumented binaries.

- `tool_results/clang_warnings.txt`
  - Clang compiler warning output (`-Wall -Wextra -Weverything`).

- `tool_results/clang_tidy_results.txt`
  - clang-tidy analysis output.

- `tool_results/cppcheck_results.txt`
  - cppcheck analysis output.

## LLM Outputs

- `llm_results/`
  - One file per benchmark program with model analysis.
  - Contains detection, identification, explanation, optimizer reasoning, severity, and fix.

## Scripts

- `scripts/run_experiments.sh`
  - Generates O0/O2 IR, assembly, and diff files for all UB benchmarks.

- `scripts/run_ubsan.sh`
  - Builds and runs UBSan/ASan binaries and records findings.

- `scripts/run_clang_warnings.sh`
  - Runs clang warning checks on benchmark files.

- `scripts/run_clang_tidy.sh`
  - Runs clang-tidy checks and logs output.

- `scripts/run_cppcheck.sh`
  - Runs cppcheck and logs output.

- `scripts/run_all_tools.sh`
  - Orchestrates all tool scripts in one command.

- `scripts/llm_query.py`
  - Batch LLM analyzer for benchmark files.
  - Uses Gemini 2.5 Flash with API key fallback and retry.

- `scripts/compare_results.py`
  - Builds final comparison tables from actual outputs.
  - Writes both CSV and Markdown summary tables.

- `scripts/make_presentation.py`
  - Generates `presentation/slides.pptx` from predefined slide content.

- `scripts/__pycache__/`
  - Python bytecode cache (auto-generated).

## Final Results and Reporting

- `results/comparison_table.csv`
  - Final machine-readable comparison table.

- `results/comparison_table.md`
  - Final human-readable comparison summary.

- `results/project_status.md`
  - Completion audit aligned with README phases.

- `report/report.md`
  - Report source text in Markdown format.

- `report/report.pdf`
  - Final report PDF output.

- `presentation/slides_outline.md`
  - Slide-by-slide presentation planning content.

- `presentation/slides.pptx`
  - Final presentation deck.

## UI

- `ui/`
  - Interactive upload-and-compare interface.

- `ui/comparison_ui.py`
  - Streamlit app.
  - Upload a C/C++ file and compare results across 5 analyzers:
    - UBSan
    - Clang warnings
    - clang-tidy
    - cppcheck
    - LLM (Gemini)
  - Displays table + clear error summaries for each tool.

## Requirements

- `requirements.txt`
  - Exact dependency snapshot exported from `.venv`.

- `requirements_setup.txt`
  - Setup commands and environment instructions.
